Complete Web Application Security Checklist for Developers

Web application security is critical in 2025. With cyber attacks becoming more sophisticated and data breaches costing millions, developers must implement comprehensive security measures. This checklist covers essential security practices every developer should follow.

1. Authentication & Authorization

✅ Implement Strong Password Policies

Require minimum 12 characters, complexity rules, and check against compromised password databases (Have I Been Pwned API).

✅ Use Multi-Factor Authentication (MFA)

Implement TOTP-based 2FA or hardware keys (WebAuthn). Never rely on SMS alone for 2FA.

✅ Secure Session Management

Use secure, httpOnly, SameSite cookies. Implement session timeout and regenerate session IDs after login.

✅ Implement Rate Limiting

Prevent brute force attacks with rate limiting on login endpoints (e.g., 5 attempts per 15 minutes).

2. Input Validation & Sanitization

✅ Validate All User Input

Never trust client-side validation alone. Implement server-side validation for all inputs using allowlists.

✅ Prevent SQL Injection

Use parameterized queries or ORMs. Never concatenate user input into SQL queries.

✅ Prevent Cross-Site Scripting (XSS)

Sanitize output, use Content Security Policy headers, and escape HTML entities in user-generated content.

✅ Prevent Command Injection

Avoid executing shell commands with user input. If necessary, use strict allowlists and escape properly.

3. CSRF Protection

✅ Implement CSRF Tokens

Add unique, unpredictable tokens to all state-changing requests (POST, PUT, DELETE).

✅ Use SameSite Cookie Attribute

Set cookies with SameSite=Strict or SameSite=Lax to prevent CSRF attacks.

✅ Verify Origin/Referer Headers

Check request origin headers for sensitive operations as an additional layer of defense.

4. Security Headers

Implement these HTTP security headers on all responses:

✅ Content-Security-Policy (CSP)

Content-Security-Policy: default-src 'self'; script-src 'self' - Prevents XSS attacks.

✅ X-Frame-Options

X-Frame-Options: DENY - Prevents clickjacking attacks.

✅ Strict-Transport-Security (HSTS)

Strict-Transport-Security: max-age=31536000; includeSubDomains - Forces HTTPS.

✅ X-Content-Type-Options

X-Content-Type-Options: nosniff - Prevents MIME type sniffing.

✅ Referrer-Policy

Referrer-Policy: strict-origin-when-cross-origin - Controls referrer information.

✅ Permissions-Policy

Permissions-Policy: geolocation=(), microphone=(), camera=() - Restricts browser features.

5. HTTPS & Transport Security

✅ Use HTTPS Everywhere

Enforce HTTPS on all pages. Redirect HTTP to HTTPS automatically.

✅ Use TLS 1.3

Disable older protocols (TLS 1.0, 1.1). Configure strong cipher suites.

✅ Implement Certificate Pinning

For high-security applications, pin SSL certificates to prevent MITM attacks.

6. Data Protection

✅ Encrypt Sensitive Data at Rest

Use AES-256 encryption for sensitive data in databases. Never store passwords in plaintext.

✅ Hash Passwords Properly

Use bcrypt, scrypt, or Argon2 with proper salt. Never use MD5 or SHA1 for passwords.

✅ Secure API Keys and Secrets

Never commit secrets to Git. Use environment variables or secret management services (AWS Secrets Manager, HashiCorp Vault).

✅ Implement Data Sanitization

Remove sensitive data before logging. Implement proper data disposal procedures.

7. API Security

✅ Use API Authentication

Implement OAuth 2.0, JWT tokens, or API keys. Never expose APIs without authentication.

✅ Implement API Rate Limiting

Prevent abuse with rate limiting (e.g., 100 requests per minute per user).

✅ Validate Content-Type

Accept only expected content types. Reject unexpected formats.

✅ Use CORS Properly

Configure CORS with specific origins. Avoid using Access-Control-Allow-Origin: * in production.

8. File Upload Security

✅ Validate File Types

Check file content, not just extensions. Use allowlists for permitted file types.

✅ Scan for Malware

Integrate antivirus scanning for uploaded files (ClamAV, cloud-based scanners).

✅ Limit File Size

Prevent DoS attacks by enforcing maximum file sizes.

✅ Store Files Securely

Store uploads outside web root or use cloud storage with proper access controls.

9. Logging & Monitoring

✅ Implement Security Logging

Log authentication attempts, authorization failures, and security events.

✅ Monitor for Anomalies

Set up alerts for suspicious activities (multiple failed logins, unusual API usage).

✅ Protect Log Files

Ensure logs don't contain sensitive data (passwords, credit cards). Restrict log access.

10. Dependency Management

✅ Keep Dependencies Updated

Regularly update npm packages, frameworks, and libraries. Use tools like Dependabot.

✅ Scan for Vulnerabilities

Run npm audit or yarn audit regularly. Fix high/critical vulnerabilities immediately.

✅ Use Software Composition Analysis (SCA)

Tools like Snyk or Sonatype help identify vulnerable dependencies.

Quick Reference Checklist

Essential Security Measures:
  • ✅ Strong authentication (MFA, password policies)
  • ✅ Input validation (SQL injection, XSS prevention)
  • ✅ CSRF protection (tokens, SameSite cookies)
  • ✅ Security headers (CSP, HSTS, X-Frame-Options)
  • ✅ HTTPS/TLS 1.3
  • ✅ Encrypted data at rest and in transit
  • ✅ Secure password hashing (bcrypt/Argon2)
  • ✅ API security (authentication, rate limiting)
  • ✅ File upload validation
  • ✅ Security logging and monitoring
  • ✅ Dependency scanning and updates
  • ✅ Regular security audits

Conclusion

Web application security requires constant vigilance and multiple layers of defense. Use this checklist as a starting point, but remember that security is an ongoing process. Regular code reviews, penetration testing, and staying updated with the latest security threats are essential.

Implement these measures progressively, prioritizing based on your application's risk profile. High-risk applications handling sensitive data should implement all measures, while lower-risk applications can focus on the most critical items first.
